Quand un mini changement de code déclenche le chaos 💥

Hier, j'ai vécu un truc de ouf avec mon app de notes. 📝

Un utilisateur m'a envoyé un message : "Hé, je peux pas créer de notes dans le dashboard !"

J'avais accidentellement changé la logique d'un hooks qui gérait les notes des utilisateurs 🙈

Voilà ce que j'ai appris de cette galère :

1. Les utilisateurs sont des ninjas 🥷
Ils trouvent toujours des façons d'utiliser ton app que t'avais même pas imaginées.

2. Le code modulaire, c'est la vie 🧩
Bouger du code sans bien tester, c'est comme jouer à la roulette russe avec ton app.

3. Écoute tes utilisateurs 👂
Leurs retours, c'est de l'or en barre pour trouver les bugs cachés.

J'ai dû faire marche arrière et tout réorganiser.

Mais vous savez quoi ?
